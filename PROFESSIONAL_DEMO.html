Validação paralela com threads

ValidationWorker — RAII thread ownership
Cada worker é um std::unique_ptr. O destructor chama join() automaticamente. workers.clear() faz join de todas as threads antes de continuar — sem detach(), sem leaks.
src/main.cpp — parallel validationC++17
std::mutex resultsMutex;
std::vector<std::unique_ptr<
    ValidationWorker>> workers;

for (const auto& node : nodes) {
  if (g_shutdown.load()) break;
  workers.push_back(
    std::make_unique<ValidationWorker>(node));
  workers.back()->run(results, resultsMutex);
}
// RAII: workers.clear() → join all threads
workers.clear();
ValidationWorker — thread + mutexC++17
void run(std::vector<ValidationResult>& results,
         std::mutex& mtx) {
  thread_ = std::thread([this, &results, &mtx]() {
    ValidationResult r = validate_node(node_);
    std::lock_guard<std::mutex> lock(mtx);
    results.push_back(std::move(r));
  });
}
// Destructor RAII:
~ValidationWorker() { join(); }

Teste de thread-safety — 20 threads
O test_validator.cpp lança 20 threads em simultâneo, cada uma chamando validate_node(), e verifica que os 20 resultados são recolhidos sem data races. Cobre o std::mutex explicitamente.
tests/test_validator.cpp — parallel testCTest
// 20 threads em simultâneo
for (const auto& node : nodes) {
  threads.emplace_back(
    [&node, &results, &mtx]() {
      auto r = validate_node(node);
      std::lock_guard<std::mutex> lock(mtx);
      results.push_back(std::move(r));
    });
}
for (auto& t : threads) t.join();
assert(results.size() == 20); // ✓
